# S-033 — Win98

## Identity

Win98 recreates the Windows 95/98 shell aesthetic: flat `#C0C0C0` silver-grey surfaces, two-pixel inset/outset bevel borders used to distinguish raised buttons from sunken input fields, navy title bars (`#000080`), and pixel-perfect MS Sans Serif typography. Every element uses the window-chrome metaphor — panels are "windows", interactive surfaces have bevel-state feedback, and scrollbars use the classic three-tone step-groove pattern. Intended for retro OS emulators, nostalgia products, developer tooling with ironic retro branding, and creative agencies signalling cultural literacy.

## "Breaks if" invariants

- breaks if `border-radius` is greater than 0px — Windows 98 had no rounded corners anywhere in the shell
- breaks if the four-tone bevel shadow is replaced with a soft drop-shadow — the four-tone inset bevel IS the Win98 button affordance
- breaks if `--motion-duration` is greater than 0ms — the OS had no CSS transitions; interactions must be instantaneous
- breaks if the title bar colour deviates from the `#000080` navy range
- breaks if a proportional serif or display font replaces the sans-serif pixel font — MS Sans Serif is the structural typeface
- breaks if `--color-bg` is replaced with white or any non-grey surface — silver grey IS the desktop chrome
- breaks if gradients are introduced — Win98 used flat fills universally
