# TECH-four-px-grid-snap

## What it does

Enforces the rule that **every coordinate, width, height, and gap in an
editorial diagram must be divisible by 4**. The rule is mechanical but
load-bearing — misaligned grids are the single biggest tell that a
diagram was generated by an AI.

## When to use

- **Every time** the skill emits HTML+SVG.
- **Every template** shipped with the skill.
- **Every worked example** in reference files.

No exceptions for small decorative elements. No exceptions for "visual
balance". No exceptions for filter padding (even the 5% filter region
pad in [primitive-sketchy](primitive-sketchy.md) is a *percentage* exception — all absolute
pixel values still snap).

## How it works

Snap-to-grid helper (pseudocode):

```javascript
function snap(v) {
  return Math.round(v / 4) * 4;
}

// Before emitting any SVG element:
const x = snap(rawX);
const y = snap(rawY);
const w = snap(rawW);
const h = snap(rawH);
```

Checking a diagram for compliance:

```javascript
// Scan all numeric coordinate attributes in the SVG
const attrs = ['x', 'y', 'width', 'height', 'cx', 'cy',
               'r', 'x1', 'y1', 'x2', 'y2', 'rx', 'ry'];

for (const el of svg.querySelectorAll('*')) {
  for (const a of attrs) {
    const v = parseFloat(el.getAttribute(a) || '0');
    if (v && v % 4 !== 0) {
      console.warn(`Off-grid: ${el.tagName}[${a}="${v}"]`);
    }
  }
}
```

Exceptions that are permitted:

- **Font sizes** (10, 11, 12, 13 px) — these stay on the type scale, not
  the 4px grid. Divisibility by 4 isn't applicable to a semantic
  attribute.
- **Stroke widths** (1, 1.5) — hairlines. These are by definition odd
  values.
- **Marker attributes** (`refX`, `refY` in arrow definitions) — these
  are relative to the marker's own viewport, not the diagram canvas.
- **Numbers inside path `d="..."` strings** — enforce when the path is
  structural (edges between nodes on a grid), but decorative curves on
  annotations can float.

## Minimal example

Compliant node:

```html
<!-- width=160, height=48, x=40, y=40 — all divisible by 4 -->
<rect x="40" y="40" width="160" height="48" rx="6"/>
```

Non-compliant node (reads as AI-generated):

```html
<!-- width=163, x=41, y=43 — off-grid; fix to 164, 40, 44 -->
<rect x="41" y="43" width="163" height="47" rx="6"/>
```

Common mistakes an LLM makes without the snap pass:

- Node widths coming out as `157`, `163`, `171` when aiming for 160
- Vertical gaps of `18`, `22`, `26` instead of 20 or 24
- Centre-aligned positions where the text-anchor is at 0.5 of a
  non-divisible width

## Gotchas

- **Text anchor vs grid.** If you centre-align text inside a 160-wide
  rect at `x=40`, the `text-anchor="middle"` coordinate is `x=120` — 120
  is divisible by 4, so no issue. If the rect had been 162 wide, the
  centre would land on 121, off-grid.
- **Spacing multiples of 4 specifically, not 8.** Some design systems use
  8px; editorial diagrams intentionally use 4px because it matches SVG
  text baseline resolution at the chosen font sizes.
- **Even `rx` values.** 6 and 10 are the only allowed border radii (6 for
  nodes, 10 for layer bands). Never 5, 7, 8, 9 — those read as jitter.
- **The filter region pad in [primitive-sketchy](primitive-sketchy.md) uses percentages
